## üìò 1. Sintaxis b√°sica de Python

Python se caracteriza por su **claridad y legibilidad**.
A diferencia de otros lenguajes, **no usa llaves `{}` ni punto y coma `;`**, sino **indentaci√≥n** para definir los bloques de c√≥digo.

### üß© Estructura m√≠nima
```python
# Esto es un comentario
def saludar(nombre):
    print(f"Hola, {nombre}!")

saludar("UNTDF")
```

### üîπ Reglas de sintaxis importantes:
- Cada bloque de c√≥digo se **indenta con 4 espacios** (no tabuladores).
- Las instrucciones terminan al final de la l√≠nea.
- Las **may√∫sculas importan** (`variable` ‚â† `Variable`).
- Los **comentarios** comienzan con `#`.

## üßÆ 2. Variables y asignaci√≥n

Una **variable** es un nombre que referencia a un valor en memoria.

### üîπ Ejemplo simple
```python
x = 10
nombre = "Ana"
pi = 3.14159
```

Python **no requiere declarar el tipo**: lo infiere autom√°ticamente (tipado din√°mico).

### üß© Reasignaci√≥n
```python
x = 10
x = "diez"
print(x)  # "diez"
```

> ‚ö†Ô∏è Esto puede causar errores si no se mantiene coherencia en los tipos.

---

### üîπ Nombres v√°lidos de variables
- Deben **comenzar con una letra o guion bajo `_`**.
- No pueden comenzar con n√∫meros.
- No deben contener espacios ni caracteres especiales.
- No deben coincidir con **palabras reservadas** (como `if`, `class`, `for`, etc.).

```python
_mi_variable = 42
edad2 = 18
# ‚ùå 2edad = 18
```

Para ver todas las palabras reservadas:
```python
import keyword
print(keyword.kwlist)
```

> üìö Referencia: [Python Lexical Analysis ‚Äì Keywords](https://docs.python.org/3/reference/lexical_analysis.html#keywords)

## üî§ 3. Tipos de datos b√°sicos en Python

Python es **din√°micamente tipado** (el tipo se infiere) y **fuertemente tipado** (no convierte tipos autom√°ticamente sin permiso).

| Tipo | Descripci√≥n | Ejemplo |
|------|--------------|----------|
| `int` | Entero | `x = 10` |
| `float` | N√∫mero decimal | `pi = 3.14` |
| `str` | Cadena de texto | `"Hola mundo"` |
| `bool` | Booleano (verdadero/falso) | `True`, `False` |
| `NoneType` | Valor nulo o vac√≠o | `None` |

---

### üîπ Ejemplos de tipos num√©ricos
```python

b = 2.5         # float
c = 2 + 3j      # complex (n√∫mero complejo)

print(a + b)    # 7.5
print(c.real)   # 2.0
print(c.imag)   # 3.0
```

> üìö Referencia: [Python Numbers](https://docs.python.org/3/library/stdtypes.html#numeric-types-int-float-complex)

## üìö 4. Estructuras de datos fundamentales

Adem√°s de los tipos simples, Python provee **estructuras compuestas** integradas:

| Tipo | Descripci√≥n | Ejemplo |
|------|--------------|----------|
| `list` | Lista mutable y ordenada | `[1, 2, 3]` |
| `tuple` | Tupla inmutable | `(1, 2, 3)` |
| `set` | Conjunto sin duplicados | `{1, 2, 3}` |
| `dict` | Diccionario clave-valor | `{"nombre": "Ana", "edad": 25}` |

---

### üîπ Listas
```python
numeros = [1, 2, 3]
numeros.append(4)
print(numeros)     # [1, 2, 3, 4]
```

### üîπ Tuplas
```python
coordenadas = (10, 20)
# coordenadas[0] = 15  ‚ùå Error: son inmutables
```

### üîπ Conjuntos
```python
colores = {"rojo", "azul", "verde", "rojo"}
print(colores)   # {'rojo', 'azul', 'verde'}
```

### üîπ Diccionarios
```python
persona = {"nombre": "Ana", "edad": 22}
print(persona["nombre"])
persona["edad"] = 23
```

## üß† 5. Conversi√≥n de tipos

A veces es necesario **convertir entre tipos**.

```python

y = int(x)     # convierte a entero
z = str(3.14)  # convierte a cadena
print(type(y), type(z))
```

> ‚ö†Ô∏è Si la conversi√≥n no es posible, Python lanza un `ValueError`.
